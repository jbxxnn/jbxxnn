## How I think about software

I am less interested in generic dashboards and more interested in:
- where work breaks down
- which decisions are still being made from memory
- where staff lose time in follow-up or coordination
- how to build the smallest system that makes the workflow clearer

The best tools usually do a few things well:
- make the next action obvious
- reduce back-and-forth
- fit the existing business behavior
- become more useful with real usage, not more complicated
